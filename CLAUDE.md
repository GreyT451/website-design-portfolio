## Key Conventions for AI Assistants

1. **Read before writing** — Always read existing files before proposing changes.
2. **Minimal changes** — Only modify what is directly needed. Avoid unnecessary refactoring.
3. **Respect existing patterns** — Follow the coding style, naming conventions, and architecture already in place.
4. **No over-engineering** — Keep solutions simple. A portfolio site should be lean and fast.
5. **Accessibility** — Use semantic HTML elements, proper ARIA attributes, and ensure keyboard navigability.
6. **Performance** — Optimize images, minimize bundle size, and avoid heavy dependencies.
7. **Responsive design** — All pages must work across mobile, tablet, and desktop viewports.
8. **Keep CLAUDE.md current** — When adding new tooling, directories, or workflows, update this file to reflect the changes.
